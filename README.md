# 🛒 買い物リスト（Notion 連携 Web アプリ）

Notion で作った買い物リストを、Web 上で見やすく整理・管理するアプリです。
カテゴリ別のグループ表示・検索・「購入済みを隠す」フィルタ、そしてチェックを入れると
**Notion 側にも書き戻し**します。

- フレームワーク: Next.js 15（App Router / TypeScript）
- Notion トークンはサーバー側（`.env.local`）に置くため、ブラウザに漏れません
- **トークン未設定でもデモデータで即プレビュー可能**

---

## セットアップ

### 1. 依存パッケージのインストール（初回のみ）

```bash
npm install
```

### 2. まずはデモで起動して見た目を確認

```bash
npm run dev
```

ブラウザで <http://localhost:3000> を開くと、サンプルの買い物リストが表示されます。

## 自分の Notion とつなぐ

### 手順 A. インテグレーション（無料）を作る

1. <https://www.notion.so/my-integrations> を開く
2. 「新しいインテグレーション」を作成
3. 表示される **Internal Integration Secret**（`secret_` または `ntn_` で始まる文字列）をコピー

### 手順 B. データベース ID を調べる

買い物リストのデータベースをブラウザで開き、URL の中の 32 文字の英数字が ID です。

```
https://www.notion.so/xxxxxxxx/<ここの32文字>?v=...
```

（ハイフンあり・なしどちらの貼り方でも動きます）

### 手順 C. データベースにインテグレーションを接続

対象のデータベースページ右上「**…**」→「**接続**（Connections）」→ 手順 A で作った
インテグレーションを追加します。**これを忘れるとデータが取得できません。**

### 手順 D. 環境変数を設定

`.env.local.example` を `.env.local` にコピーし、値を入れます。

```bash
cp .env.local.example .env.local
```

```env
NOTION_TOKEN=secret_xxxxxxxxxxxxxxxxxxxx
NOTION_DATABASE_ID=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 手順 E. 再起動

```bash
npm run dev
```

デモ表示が消え、あなたの Notion のデータが並べば成功です。

## Notion データベースの列（プロパティ）について

列名は自由ですが、以下を用意すると自動で認識します（**型で判定**するので日本語・英語どちらでも可）。

| 役割        | 対応する Notion の型         | 認識される列名の例                     |
| ----------- | --------------------------- | -------------------------------------- |
| 品名        | タイトル（必須）             | 名前 / 品名 / Name                     |
| カテゴリ    | セレクト / ステータス        | カテゴリ / 分類 / 種類 / Category      |
| 購入済み    | チェックボックス             | 完了 / 購入済み / Done                 |
| 数量        | 数値 / テキスト              | 数量 / 個数 / Quantity                 |
| メモ        | テキスト                     | メモ / 備考 / Note                     |
| リンク      | URL                          | （URL 型なら何でも）                   |

> 上記に当てはまらない列も、各アイテムの下に「列名: 値」として表示されます。
> チェックの書き戻しには、DB に**チェックボックス型の列**が 1 つ必要です。

## 本番ビルド

```bash
npm run build
npm start
```

## 補足

- `.env.local` は `.gitignore` 済み。トークンが誤って公開されることはありません。
- データは毎回 Notion から最新を取得します（キャッシュしません）。
