# 🛒 買い物リスト（Notion 連携 Web アプリ）

Notion で作った買い物リストを、Web 上で見やすく整理・管理するアプリです。
カテゴリ別のグループ表示・検索・「購入済みを隠す」フィルタ、そしてチェックを入れると
**Notion 側にも書き戻し**します。

- フレームワーク: Next.js 15（App Router / TypeScript）
- Notion トークンはサーバー側（`.env.local`）に置くため、ブラウザに漏れません
- **トークン未設定でもデモデータで即プレビュー可能**

---

## セットアップ

### 1. 依存パッケージのインストール（初回のみ）

```bash
npm install
```

### 2. まずはデモで起動して見た目を確認

```bash
npm run dev
```

ブラウザで <http://localhost:3000> を開くと、サンプルの買い物リストが表示されます。

## 自分の Notion とつなぐ

### 手順 A. インテグレーション（無料）を作る

1. <https://www.notion.so/my-integrations> を開く
2. 「新しいインテグレーション」を作成
3. 表示される **Internal Integration Secret**（`secret_` または `ntn_` で始まる文字列）をコピー

### 手順 B. データベース ID を調べる

買い物リストのデータベースをブラウザで開き、URL の中の 32 文字の英数字が ID です。

```
https://www.notion.so/xxxxxxxx/<ここの32文字>?v=...
```

（ハイフンあり・なしどちらの貼り方でも動きます）

### 手順 C. データベースにインテグレーションを接続

対象のデータベースページ右上「**…**」→「**接続**（Connections）」→ 手順 A で作った
インテグレーションを追加します。**これを忘れるとデータが取得できません。**

### 手順 D. 環境変数を設定

`.env.local.example` を `.env.local` にコピーし、値を入れます。

```bash
cp .env.local.example .env.local
```

```env
NOTION_TOKEN=secret_xxxxxxxxxxxxxxxxxxxx
NOTION_DATABASE_ID=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 手順 E. 再起動

```bash
npm run dev
```

デモ表示が消え、あなたの Notion のデータが並べば成功です。

## Notion データベースの列（プロパティ）について

列名は自由ですが、以下を用意すると自動で認識します（**型で判定**するので日本語・英語どちらでも可）。

| 役割        | 対応する Notion の型         | 認識される列名の例                     |
| ----------- | --------------------------- | -------------------------------------- |
| 品名        | タイトル（必須）             | 名前 / 品名 / Name                     |
| カテゴリ    | セレクト / ステータス        | カテゴリ / 分類 / 種類 / Category      |
| 購入済み    | チェックボックス             | 完了 / 購入済み / Done                 |
| 数量        | 数値 / テキスト              | 数量 / 個数 / Quantity                 |
| メモ        | テキスト                     | メモ / 備考 / Note                     |
| リンク      | URL                          | （URL 型なら何でも）                   |

> 上記に当てはまらない列も、各アイテムの下に「列名: 値」として表示されます。
> チェックの書き戻しには、DB に**チェックボックス型の列**が 1 つ必要です。

## 会場で使う（デプロイ & オフライン）

外出先（同人イベント会場など）でスマホから使うには、どこかに公開する必要があります。**Vercel（無料）** が最も簡単です。

### 1. GitHub に push

```bash
# すでに git init 済み。GitHub で空リポジトリを作り、URL を控える
git remote add origin https://github.com/<あなた>/<リポジトリ名>.git
git branch -M main
git push -u origin main
```

`.env.local`（トークン入り）は `.gitignore` 済みなので push されません。

### 2. Vercel にインポート

1. <https://vercel.com> に GitHub でログイン → 「Add New… → Project」
2. 先ほどのリポジトリを選択（フレームワークは Next.js が自動認識）
3. **Environment Variables** に以下を登録:
   | Key | Value |
   | --- | --- |
   | `NOTION_TOKEN` | インテグレーションのトークン |
   | `NOTION_DATABASE_ID` | データベース ID |
   | `APP_PASSWORD` | 好きな合言葉（アクセス保護。**公開時は必ず設定推奨**） |
4. 「Deploy」→ 数十秒で `https://xxxx.vercel.app` が発行される

以降は `git push` するたびに自動で再デプロイされます。

### 3. スマホに入れる

- 発行された URL をスマホで開く（`APP_PASSWORD` を設定した場合はログイン画面。ユーザー名は任意、パスワードに合言葉）
- ブラウザメニューから **「ホーム画面に追加」**（PWA）→ アプリのように起動できる

### 4. 会場の電波対策（オフライン対応）

大規模会場はモバイル回線が混雑しがちです。本アプリはオフラインに対応しています:

- **入場前など電波のある場所で一度開く** → リストが端末にキャッシュされる
- 会場では**オフラインでも閲覧・チェック操作が可能**（チェック/状態変更は端末に保存）
- ヘッダーに **「● オフライン」「未同期 N」** を表示。**電波が戻ると自動でまとめて Notion に同期**（手動同期は ↻ ボタン）
- ホーム画面から起動すればアプリ本体もキャッシュから立ち上がる

> コツ: 会場入り口など電波のあるうちに一度アプリを開いて最新化しておくと安心です。

## 補足

- `.env.local` は `.gitignore` 済み。トークンが誤って公開されることはありません。
- `APP_PASSWORD` 未設定なら認証なし（ローカル開発向け）。公開環境では必ず設定してください。
- オフライン用データはブラウザの localStorage に保存されます（端末ごと）。
